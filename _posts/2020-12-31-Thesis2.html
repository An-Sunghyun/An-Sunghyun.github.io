[Thesis] Exploratory research based on big data for Improving the revisit rate of foreign tourists and invigorating consumption

Rstudio was used for this thesis

Problem Definition

Satisfaction cannot always be high when foreign tourists eat at a restaurant that Koreans like. This is because there are differences in individual tendencies and cultures. If so, I think it will greatly increase satisfaction and improve convenience if you can guide them to a restaurant that fits their tendencies in public and universally and recognized by Koreans. Therefore, the problem of this study is to be defined as "how to find and utilize the commonalities of restaurants that foreigners and Koreans think of."

Data Set

TripAdvisor

Naver

In order to analyze the commonalities between the restaurants that foreigners like and the restaurants that Koreans like, it was first necessary to comment on the national site of foreigners who finished sightseeing. To this end, foreign restaurant reviews were collected using a review site called Tripadvisor, and Korean reviews were collected from Naver, the most commonly used search engine in Korea. It is necessary to analyze this data to find information on foods and restaurants that foreigners and Koreans are satisfied with.

Analysis techniques for deriving information

In order to find out the preferred food and taste from the previously collected data, important words were extracted through latent Dirichlet allocation (LDA) analysis and classified by related topics to derive results. In addition, additional text mining was conducted to find restaurants or food preferred by both Koreans and foreigners, and this was supplemented by the circular graph of visualization and the word cloud. Analysis first collects raw data and purifies it to enable analysis through preprocessing. Thereafter, desired information is calculated through LDA analysis and text mining using the analysis tool(RStudio)

Conclusion

Deep learning models implemented by tensorflow have a canonical graph, but pytorch has not. I think input shape is the main reason why pytorch didn't work properly. I can set input shape with tensorflow, but pytorch can't. I gathered 48X48 image data and I could set tensorflow's input shape as 48X48. But I couldn't set pytorch's input shape so I had no choice but to train model with 256x256 size. The data I have is 48X48, but the input shape was 256X256, so I think it cannot have been learned properly. VGG-16 was the best result with 89% validation acc. However, it is hard to conclude that it is the best because it is a narrow gap with densenet and resnet.
