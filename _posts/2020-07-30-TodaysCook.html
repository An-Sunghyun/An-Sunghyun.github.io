[Website] Today's Cook

Sites where you can search for recipes through your current ingredients or get menu recommendations through fragmentary conditions for what you want to eat.

In addition, you can search for detailed information about each food.

Design

All design components were made directly through Photoshop.

Main Cover

Navbar

  1. Home logo: Go to main page
  2. Cooking search: Go to the main page search window
  3. Recommend recipe: Go to the recipe search page
  4. Menu Recommendation: Go to Menu Search page
  5. Design: Implemented responsively using CSS

Main Page

Today's Recommend

  1. Today's recommended menu: Select one randomly from the menu and print out the information
  2. Today's recommended recipe: Select one randomly from the recipe and print out the information

Main Page Bottom Button

  1. Menu Recommendation Button: Go to Menu Search page
  2. Recommend recipe button: Go to the recipe search page
  3. Design: CSS is used to produce a reactive type. Apply Hover Effect

Main Page Search

  1. Write the name of the food correctly
  2. Go to the food search results page and print the results
  3. Search the database and print the name, unit calories, taste, and section information in the form of photos and tables.

Menu Search

  1. Conditions of selection are implemented as radio boxes (single selection)
  2. Go to the Menu Search Results page through the Submit button
  3. Insert the selected item as a variable value into the SQL statement and search the database
  4. Today's recommended menu/recipe information is displayed on the right side

Recipe Search

  1. Implementation of selection conditions with check boxes (multiple selections, up to eight selections are limited)
  2. Use the Submit button to go to the recipe search results page.
  3. Insert the selected item as a variable value into the SQL statement and search the database
  4. Today's recommended menu/recipe information is displayed on the right side

Food Search Result

  1. Information about the food searched
  2. Information on food names, calories per unit, taste, and large category is displayed.
  3. Information is expressed in a table format.
  4. Printed with photos for visual effects
  5. Today's recommended menu/recipe information is displayed on the right side.

Menu Search Result

  1. Search the database via SQL statements using the conditions entered on the Menu Search page.
  2. Select up to three results that meet the conditions and print them out in the form of a card title.
  3. At the bottom, go to the relevant search page through the menu re-search and recipe re-search buttons.

Recipe Search Result

  1. Search the database via SQL statements using the conditions entered on the Recipe Search page.
  2. Select up to three results that meet the conditions and print them out in the form of a card title.
  3. In the recipe results, click on the results and link them to the recipe site related to the dish.
  4. Go to the search page at the bottom through the 'Rescan Menu' and 'Rescan Recreate Recipe' buttons

Backend

Search Function

  1. Enter the name of the food you want to search for in the search window on the main page and click the Enter or Submit button.
  2. Invoke the corresponding Result Servlet as the event occurred
  3. Servlet extracts the name of food entered in the search box with data and calls DAO through Service
  4. Request that the recalled DAO search the database for the name of the food received as data in SQL statements.
  5. If data of that food name exists in the database, the database sends it to DAO.
  6. Return transmitted food data to a servlet by storing it in the DAO's list using ArrayList
  7. Visualize search results by invoking the results screen JSP for the function that corresponds with the returned list

Menu Search Recommend Function

  1. Go to the Select Menu Recommendation Criteria page through the menu recommendation button, etc.
  2. Select a filter through the radio button and click the search button
  3. Invoke the corresponding MenuResult Servlet as an event occurred
  4. The servlet extracts data from selected conditions and calls the DAO through the service.
  5. Request the database to search for conditions received by the invoked DAO in SQL statements
  6. If data corresponding to the conditional data exists in the database, the database sends it to DAO.
  7. Data received is stored in the DAO's ArrayList list and returned as a servlet
  8. Visualize search results by invoking the results screen JSP of the function that corresponds to the returned list.

Recipe Search Recommend Function

  1. Go to the Select Recipe Recommendation Criteria page through the recipe recommendation button, etc.
  2. Select duplicate selections (up to 8) for Checkbox and click the Search button.
  3. Invoke the corresponding RecipeResult Servlet as an event occurred
  4. The servlet extracts data from selected conditions and calls the DAO through the service.
  5. Request the database to search for conditions received by the invoked DAO in SQL statements
  6. If the data corresponding to the conditional data resides in the database, the database sends it to DAO.
  7. Data received is stored in the DAO's ArrayList list and returned as a servlet
  8. Visualize search results by invoking the results screen JSP of the function that corresponds to the returned list.

Today's Recommend Function

  1. When the main page opens, back-end automatically requests a random search and transfer of one menu/recipe data from the database.
  2. Transferring menus/rescriptions from the database to the backend one by one
  3. Store data sent in a back-end, ArrayList-enabled list, and time-racked data
  4. On each page with today's recommended card side, data stored in the list is inserted as a result to visualize the results.
